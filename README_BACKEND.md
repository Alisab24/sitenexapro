# Architecture Backend SaaS - NexaPro → LeadQualif

## 🏗️ Vue d'ensemble

Cette architecture connecte le formulaire de contact de nexapro.tech directement au pipeline LeadQualif via une Edge Function Supabase sécurisée.

```
Formulaire nexapro.tech 
  → POST Supabase Edge Function "add-lead" 
  → Validation + anti-spam + enrichissement 
  → INSERT table leads Supabase 
  → Lead visible dans pipeline LeadQualif
```

## 🔐 Sécurité implémentée

### 1. **CORS restrictif**
- Origins autorisées : `https://nexapro.tech`, `https://www.nexapro.tech`
- Méthodes autorisées : `POST, OPTIONS`
- Headers autorisés : `Content-Type, x-api-key`

### 2. **API Key personnalisée**
- Clé `NEXAPRO_API_KEY` requise dans header `x-api-key`
- Stockée dans secrets Supabase (jamais dans le code)

### 3. **Rate limiting**
- 5 requêtes maximum par IP par heure
- Map en mémoire (remplacer par Redis en production)
- Protection contre DDoS et spam

### 4. **Validation stricte**
- Validation client-side et server-side
- Longueurs maximales des champs
- Regex email stricte
- Anti-spam honeypot

### 5. **Row Level Security (RLS)**
- Seul `service_role` peut insérer des leads
- Utilisateurs authentifiés voient seulement leurs leads (multi-tenant)
- Politiques granulaires par table

## 📊 Schéma de la table `leads`

```sql
CREATE TABLE leads (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  first_name TEXT,
  last_name TEXT,
  name TEXT,
  email TEXT NOT NULL,
  company TEXT,
  profile TEXT,
  message TEXT,
  source TEXT DEFAULT 'direct',
  status TEXT DEFAULT 'new',
  score INTEGER DEFAULT 0,
  priority TEXT DEFAULT 'normal',
  ip_address TEXT,
  tenant_id TEXT DEFAULT 'nexapro-default',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Index pour performance
CREATE INDEX idx_leads_email ON leads(email);
CREATE INDEX idx_leads_tenant ON leads(tenant_id);
CREATE INDEX idx_leads_status ON leads(status);
CREATE INDEX idx_leads_source ON leads(source);
```

## 📦 Déploiement

Voir `DEPLOY.md` pour les commandes complètes :

```bash

# 4. Appliquer les migrations
```

## ✅ Points de vérification

- [ ] La service_role key n'apparaît NULLE PART dans le code frontend
- [ ] La Edge Function rejette les requêtes sans x-api-key valide
- [ ] La Edge Function rejette les origines non listées dans ALLOWED_ORIGINS
- [ ] Le honeypot est invisible dans le DOM (position absolute, hors écran)
- [ ] Le rate limiting bloque après 5 soumissions par IP par heure
- [ ] Les leads créés depuis nexapro.tech ont source="nexapro-website" dans Supabase
- [ ] RLS est activé sur la table leads
- [ ] Le fichier .env.local est dans .gitignore
- [ ] Le formulaire affiche un message de succès sans révéler d'infos techniques
